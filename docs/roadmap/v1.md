# Aipany V1 Roadmap

## V1 objective

Ship a mobile-first realtime AI voice companion that feels natural in continuous conversation while proving the reusable platform architecture needed for future ESP32-S3 clients.

## Phase 0 — Foundation

Deliverables:

- pnpm + Turborepo monorepo;
- shared TypeScript configuration;
- versioned device/session protocol;
- architecture decision documentation;
- initial issue backlog.

Exit criteria: the repository has a stable structure and development tasks can proceed independently without redefining core boundaries.

## Phase 1 — Realtime voice vertical slice

Deliverables:

- React Native mobile app shell;
- authentication placeholder or development identity;
- microphone permission and audio session handling;
- realtime session bootstrap endpoint;
- WebRTC connection to the first realtime voice provider;
- remote audio playback;
- connection and session state UI;
- explicit start/end session controls.

Exit criteria: a user can open the app and hold a live two-way voice conversation.

## Phase 2 — Human-like interaction quality

Deliverables:

- server-side session policy;
- semantic turn detection where supported;
- reliable barge-in and response cancellation;
- playback truncation state handling;
- realtime transcript events;
- latency instrumentation for speech start, first audio, interruption, and tool calls;
- reconnect and degraded-network behavior.

Exit criteria: users can naturally pause, interrupt, correct themselves, and continue without push-to-talk behavior.

## Phase 3 — AI Brain

Deliverables:

- Agent Service;
- configurable persona and voice policy;
- Tool Service with a provider-neutral tool registry;
- deep-task delegation interface;
- short conversational acknowledgements before long-running tools;
- tool execution audit trail.

Exit criteria: realtime voice handles conversation while complex work can run outside the realtime model and return naturally to the same session.

## Phase 4 — Memory

Deliverables:

- PostgreSQL persistence;
- Memory Service;
- memory extraction pipeline;
- memory retrieval before or during sessions;
- user-editable and deletable memories;
- conversation summaries;
- privacy and retention controls.

Exit criteria: the assistant remembers useful information across sessions without blindly replaying complete chat history.

## Phase 5 — Productization

Deliverables:

- account system;
- session history;
- agent configuration UI;
- usage metrics;
- rate limits and quotas;
- operational logging and error reporting;
- production secret management.

Exit criteria: the app can support external beta users safely and observably.

## Phase 6 — Embedded readiness

Deliverables:

- `clients/esp32-sdk` protocol adapter design;
- device provisioning flow;
- embedded authentication design;
- capability-based device commands;
- firmware/OTA contracts;
- reference ESP32-S3 transport prototype.

Exit criteria: an ESP32 client can join the same platform without changing Agent, Memory, Tool, or Billing service contracts.

## V1 quality metrics

Track at minimum:

- session establishment time;
- end-of-user-turn to first assistant audio;
- user-speech-start to assistant-playback-stop during interruption;
- reconnect success rate;
- tool-call latency;
- realtime session failure rate;
- cost per active voice minute.
