# Aipany V1 开发路线图

## V1 目标

发布一个以手机端为起点的实时 AI 语音陪伴应用，让连续对话足够自然，同时验证未来接入 ESP32-S3 所需要的可复用平台架构。

## 阶段 0 —— 基础架构

交付内容：

- pnpm + Turborepo Monorepo；
- 统一 TypeScript 配置；
- 带版本号的设备 / 会话协议；
- 系统架构决策文档；
- 第一批开发任务。

完成标准：仓库结构稳定，不同模块可以独立开发，不需要反复重新定义核心边界。

## 阶段 1 —— 实时语音最小闭环

交付内容：

- React Native 手机 App 基础工程；
- 临时认证或开发环境身份；
- 麦克风权限和 Audio Session 管理；
- 实时语音 Session 创建接口；
- 与第一家实时语音供应商建立 WebRTC 连接；
- AI 远端语音播放；
- 连接状态和 Session 状态 UI；
- 明确的开始 / 结束会话控制。

完成标准：用户打开 App 后，可以与 AI 进行一次完整的双向实时语音对话。

## 阶段 2 —— 真人感交互质量

交付内容：

- 服务端 Session Policy；
- 供应商支持时启用 Semantic Turn Detection；
- 稳定的 Barge-in 打断和回复取消；
- AI 未播放内容的截断状态处理；
- 实时字幕事件；
- 语音开始、首包音频、打断、Tool Call 等延迟指标；
- 断线重连和弱网降级策略。

完成标准：用户可以自然停顿、打断 AI、纠正自己并继续交流，不需要 Push-to-Talk。

## 阶段 3 —— AI Brain

交付内容：

- Agent Service；
- 可配置的人设和声音策略；
- Provider 无关的 Tool Registry；
- 深度任务委派接口；
- 长任务执行前的自然口头回应；
- Tool 执行审计记录。

完成标准：实时语音模型专注于持续交流，复杂任务可以在实时模型之外执行，并自然返回到同一个语音会话。

## 阶段 4 —— 长期记忆

交付内容：

- PostgreSQL 持久化；
- Memory Service；
- 记忆提取流程；
- 会话前或会话中的相关记忆检索；
- 用户可查看、修改和删除的记忆；
- 对话摘要；
- 隐私与保留策略。

完成标准：AI 可以跨会话记住有价值的信息，而不是每次简单重放全部历史聊天记录。

## 阶段 5 —— 产品化

交付内容：

- 用户账号系统；
- 会话历史；
- Agent 配置界面；
- 用量统计；
- 限流和配额；
- 运行日志和错误监控；
- 生产环境密钥管理。

完成标准：App 可以安全、可观测地开放给外部 Beta 用户使用。

## 阶段 6 —— 嵌入式设备准备

交付内容：

- `clients/esp32-sdk` 协议适配设计；
- 设备配网与注册流程；
- 嵌入式设备认证方案；
- 基于 Capability 的设备命令；
- 固件 / OTA 协议；
- ESP32-S3 实时传输参考原型。

完成标准：ESP32 客户端可以加入同一平台，而不需要修改 Agent、Memory、Tool 或 Billing 的核心服务协议。

## V1 核心质量指标

至少持续记录：

- Session 建立耗时；
- 用户一轮语音结束到 AI 第一段音频播放的耗时；
- 用户开始打断到 AI 本地播放停止的耗时；
- 断线重连成功率；
- Tool Call 延迟；
- 实时 Session 失败率；
- 每分钟有效语音对话成本。
